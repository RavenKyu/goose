get방식의 parameter값을 이용하여 json데이터를 출력한다.
table : db table name
mod : 모듈id
group : 모듈그룹 번호
act : index|single|put|edit|delete (목록, 한개, 입력, 수정, 삭제)
idx : 글 고유번호
order : srl (정렬기준이 되는 필드)
sort : asc|desc
limit : 10
page : 1
searchField : 검색필드
searchKeyword : 검색어
field : 출력 필드

put, edit, delete는 article 테이블만으로 제한한다.
put, edit, delete 기능은 좀 생각해보고 추후에 넣어볼 생각이다.
act=single 기능은 데이터 하나만 조회가 가능하다. 고민고민 하다가 혹시 몰라서 key,value 값 받기로 했다.
api인증방식에 대해 고민해보고 적용시켜야한다.
xml이나 json으로 변환이 쉽기 때문에 파일을 나누는것보다 다른 파일이름으로 변경하는게 좋아보인다.

// 2014. 5. 29
일단 apikey인증은 프로그램 고정으로 만들어뒀다. 회원관리 기능이 전혀없기 때문에 회원에 따른 key값을 넣자고 생각은 해봤지만 관뒀다.
/api/json/ 경로는 json전용인 방식인듯하기도 하고 xml로 변경이 쉽기 때문에 크게 나눌 필요가 없어 보인다. 그래서 이름을 item으로 변경했다. 데이터 입력은 put, 수정은 edit, 삭제는 delete로 액션에 따른 파일로 나눠야겠다고 보인다. rss도 만들어볼까하는 생각도 든다. ㅎㅎ~
